# 管理台前端架构设计

## 设计目标

管理台是电子相册内容维护工具，界面按“相册管理”和“计划管理”组织。用户主要在手机上使用，界面按手机 App 管理工具设计，使用顶部 App Bar、底部模块导航、浮动主操作、素材格、计划列表和底部弹窗表单。

视觉约束：

- 圆角控制在 `2px` 到 `4px`。
- 主要使用分隔线和表格化排布。
- 移动端控件满足触控尺寸，按钮只承载明确命令。
- 上传、编辑、删除确认都通过弹窗完成。
- 页面常态只展示数据，不常驻保存按钮和大表单。

`ConsolePage.vue` 只负责模块切换。`views/` 承载控制台一级视图的数据加载和操作入口，`components/` 承载可复用基础控件和业务局部组件。

基础组件按职责分层：

- `base/` 放最基础命令控件，当前只有 `Button.vue`。按钮同时支持文字、图标和纯图标按钮。
- `input/` 放输入控件，日期使用自绘 `DatePicker.vue`，不使用浏览器原生 `type=date`。
- `display/` 放展示原子组件，当前为 `Icon.vue`。
- `feedback/` 放空状态等反馈组件。
- `navigation/` 放操作菜单。
- `overlay/` 放弹窗和弹窗动作区。

`components/index.ts` 统一导出通用组件和必要类型。业务局部组件可以按相对路径直连领域目录，页面和布局优先从统一入口导入通用控件。

## 页面框架

移动端使用 App 结构：

```text
电子相册管理台                                      管理员  退出

内容区

相册        计划
```

内容区只显示当前模块，不同时堆叠相册和计划。默认进入相册管理。底部导航固定在页面底部，桌面端只作为宽屏适配展开尺寸，不另起一套导航。上传图片和新增计划是当前模块的主操作，移动端使用右下角浮动按钮，桌面端放在模块工具条内。

## 相册管理

图片数据只有 `sha256`、`status`、`remark`，相册管理只围绕这三个字段和 display 预览设计。

顶部工具条：

```text
相册管理    搜索备注或 sha256    状态筛选    查询    上传图片
```

移动端工具条只承载搜索、筛选和查询，上传图片作为浮动主操作。

主体使用 5:3 素材格：

- `ready` 显示 display 预览。
- `pending` 显示“待处理”占位。
- `processing` 显示“处理中”占位。
- `failed` 显示“处理失败”占位，并用细红色边线标记。

每个素材格只显示备注、短 sha、状态和图标操作菜单。菜单包含：

- 编辑备注。
- 刷新预览，仅 `ready` 图片显示。

上传图片通过弹窗完成，页面不铺上传表单。编辑备注通过弹窗完成，页面不常驻备注输入框和保存按钮。

## 计划管理

计划数据由 `date`、`caption`、`image` 组成，计划管理围绕计划日期、标题和单张图片编排。

顶部工具条：

```text
计划管理    最近 N 天    查询    新增计划
```

移动端工具条只承载范围筛选和查询，新增计划作为浮动主操作。

计划管理主体使用列表项，每条计划以 `date` 为唯一记录，列表项展示日期、标题、单张图片缩略图、图片 sha256 和图标操作菜单：

```text
日期      标题        图片          图片 SHA256      操作
06-09     晚风和海    [图]          0000...0000      图标菜单
```

菜单包含：

- 编辑。
- 删除。

移动端列表项顶部展示日期、标题和操作入口，内容区展示单张图片缩略图和短 SHA。

新增和编辑计划都通过弹窗完成。计划弹窗包含标题、计划日期和图片选择器。新增计划默认日期为当天。计划日期使用统一日期选择组件，组件内部渲染日期弹窗，输出值固定为 `YYYY-MM-DD`。

图片选择器使用紧凑缩略格：

- 一个计划只能选择一张图片。
- 点击图片选中，再次点击取消。
- 选中态用细边框标记。
- 未处理完成的图片可以选择，但必须显示状态。
- 底部只显示“已选 1 张”或“未选图片”。

删除计划通过确认弹窗完成。

## 请求与状态

API 按资源拆分：

- `api/auth.ts`
- `api/images.ts`
- `api/plans.ts`
- `api/client.ts`
- `api/types.ts`

登录 token 使用 `useAuthStore` 统一管理。相册管理和计划管理组件各自加载自己需要的数据，避免页面组件承载所有业务状态。

## 验收标准

- `App.vue` 只负责登录态分流。
- `ConsolePage.vue` 只负责模块切换。
- 移动端使用底部模块导航，默认进入相册管理。
- 图片上传、备注编辑、计划编辑和删除确认都在弹窗中完成。
- 移动端上传图片和新增计划使用右下角浮动主操作。
- 相册管理使用紧凑素材格承载图片状态、备注和操作入口。
- 计划管理使用日期、标题、缩略图和操作入口组成的列表项。
- 一个计划在前端只能选择一张图片。
- 计划日期使用自绘日期选择组件，不依赖原生日期输入。
- 编辑计划时，更新接口路径中的 `date` 表示原计划日期，body 中的 `date` 表示保存后的计划日期。
- 相册管理和计划管理的操作入口使用图标菜单。
- 全局不存在重复刷新按钮。
- `bun run build` 通过。
