# 管理台前端架构设计

## 设计目标

管理台是电子相册内容维护工具，界面按“图片素材库”和“显示排期”组织。页面保持工具型风格，使用顶部工作区切换、紧凑工具条、素材格、排期条和弹窗表单。

视觉约束：

- 圆角控制在 `2px` 到 `4px`。
- 主要使用分隔线和表格化排布。
- 控件尺寸偏小，按钮只承载明确命令。
- 上传、编辑、删除确认都通过弹窗完成。
- 页面常态只展示数据，不常驻保存按钮和大表单。

## 页面框架

顶部固定为工作区切换：

```text
电子相册管理台    概览  图片库  计划管理                  管理员  退出
```

内容区只显示当前工作区，不同时堆叠图片和计划。

## 图片库

图片数据只有 `sha256`、`status`、`remark`，图片库只围绕这三个字段和 display 预览设计。

顶部工具条：

```text
图片库    搜索备注或 sha256    状态筛选    查询    上传图片
```

主体使用 5:3 素材格：

- `ready` 显示 display 预览。
- `pending` 显示“待处理”占位。
- `processing` 显示“处理中”占位。
- `failed` 显示“处理失败”占位，并用细红色边线标记。

每个素材格只显示备注、短 sha、状态和图标操作菜单。菜单包含：

- 编辑备注。
- 刷新预览，仅 `ready` 图片显示。

上传图片通过弹窗完成，页面不铺上传表单。编辑备注通过弹窗完成，页面不常驻备注输入框和保存按钮。

## 计划管理

计划数据由 `date`、`caption`、`image_sha256` 组成，计划管理围绕计划日期、标题和单张图片编排。

顶部工具条：

```text
计划管理    最近 N 天    查询    新增计划
```

主体使用计划记录表，每条计划以 `date` 为唯一记录，在计划表中展示一行：

```text
日期      标题        图片          图片 SHA256      操作
06-09     晚风和海    [图]          0000...0000      图标菜单
```

计划行只显示日期、标题、单张图片缩略图、图片 sha256 和图标操作菜单。菜单包含：

- 编辑。
- 删除。

新增和编辑计划都通过弹窗完成。计划弹窗包含标题、计划日期和图片选择器。计划日期使用统一日期输入基础组件。

图片选择器使用紧凑缩略格：

- 一个计划只能选择一张图片。
- 点击图片选中，再次点击取消。
- 选中态用细边框标记。
- 未处理完成的图片可以选择，但必须显示状态。
- 底部只显示“已选 1 张”或“未选图片”。

删除计划通过确认弹窗完成。

## 概览

概览只读展示，不放业务表单：

- 图片总数。
- 可显示图片数。
- 处理中图片数。
- 失败图片数。
- 计划数量。
- 已配置图片的计划数。
- 最近计划摘要。

## 请求与状态

API 按资源拆分：

- `api/auth.ts`
- `api/images.ts`
- `api/plans.ts`
- `api/client.ts`
- `api/types.ts`

登录 token 使用 `useAuthStore` 统一管理。图片库和计划管理组件各自加载自己需要的数据，避免页面组件承载所有业务状态。

## 验收标准

- `App.vue` 只负责登录态分流。
- `ConsolePage.vue` 只负责工作区切换。
- 图片上传、备注编辑、计划编辑和删除确认都在弹窗中完成。
- 图片库使用素材格，不使用大卡片列表。
- 计划管理使用计划记录表，不使用日期分组和大卡片列表。
- 一个计划在前端只能选择一张图片。
- 计划日期输入使用统一日期组件。
- 编辑计划时，更新接口路径中的 `date` 表示原计划日期，body 中的 `date` 表示保存后的计划日期。
- 图片库和计划管理的操作入口使用图标菜单。
- 全局不存在重复刷新按钮。
- `bun run build` 通过。
